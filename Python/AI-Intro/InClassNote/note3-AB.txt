对于AB剪枝的MiniMax的二次深入学习：
一句话概括————【MAX节点更新传入的alpha，MIN节点更新传入的beta，另外一个原值传递，beta <= alpha的时候就剪枝】
最妙的一点就是这个剪枝条件【beta <= alpha】是“对称成立”的，也就是对MIN/MAX节点都是满足这个条件就剪枝


【定义】
在 α-β 剪枝中，每个节点携带两个边界值（从祖先继承）：
α：MAX 节点保证能得到的最低分数（下界）。对于 MAX 节点，α 初始为 -∞，随着搜索不断更新为已发现的最大值；
对于 MIN 节点，α 继承自父节点的 α，表示 MAX 在父节点已经确保至少能达到 α。

β：MIN 节点能限制 bot 得到的最高分数（上界）。对于 MIN 节点，β 初始为 +∞，随着搜索不断更新为已发现的最小值；
对于 MAX 节点，β 继承自父节点的 β，表示 MIN 在父节点已经保证 bot 不会超过 β。
关键：在搜索过程中，如果某一层的 α >= β，就说明当前节点的某个分支已经足以让对方拒绝选择其他分支，可以立即剪枝。


【启发函数排序：加速剪枝】
这里先对每个动作进行一次浅层评估，得到 score，然后根据 isMaximizing 对 scoredMoves 排序：
MAX 节点：降序（bot 优先尝试自己认为好的动作）
MIN 节点：升序（优先尝试人类可能选择的让 bot 得分低的动作）
排序目的：α-β 剪枝的效率高度依赖节点访问顺序。
如果先访问最有利（对当前节点而言）的动作，就能更快地收紧边界，从而剪掉更多分支。这是典型的启发式排序。


【为什么alpha <= beta这一剪枝条件对MAX和MIN节点均成立？】
剪枝条件的直观理解：

在 MAX 节点：如果某个子节点返回的评估值 eval 已经 ≥ β，
那么父 MIN 节点（因为父节点会取 min）就不会选择这个子节点所在的路径
（因为父节点已经有另一个子节点的值 ≤ β，而这里 ≥ β 的路径不会被选）。
因此，在 MAX 节点中，一旦 eval 足够大使得 alpha ≥ β，后续子节点就不再需要搜索。

在 MIN 节点：如果某个子节点返回的评估值 eval 已经 ≤ α，
那么父 MAX 节点（会取 max）就不会选择这个子节点所在的路径
（因为父节点已经有另一个子节点的值 ≥ α，而这里 ≤ α 的路径不会被选）。
因此，在 MIN 节点中，一旦 eval 足够小使得 beta ≤ α，后续子节点就不再需要搜索。

MAX节点： MIN           MIN节点： MAX
        /    \                  /    \
      MAX    MAX              MIN    MIN
      / \    / \             / \    / \
    MIN MIN MIN MIN         MAX MAX MAX MAX
**MAX节点父为MIN，子为MIN；MIN节点父为MAX，子为MAX**

在 MAX 节点：更好的结果是指更大的评估值。如果当前节点已经找到一个值 ≥ β（父 MIN 节点的上界），
那么后续任何子节点返回的值只会更大（或相等），无法让父 MIN 节点改变选择（父 MIN 会选最小的），因此剪枝。
在 MIN 节点：更好的结果是指更小的评估值。如果当前节点已经找到一个值 ≤ α（父 MAX 节点的下界），
那么后续任何子节点返回的值只会更小（或相等），无法让父 MAX 节点改变选择（父 MAX 会选最大的），因此剪枝。

所以，无论当前节点是 MAX 还是 MIN，
一旦发现当前分支已经无法提供比祖先节点已确定界限“更好”（即对祖先节点更有吸引力）的结果，
就停止搜索该节点的剩余子节点。这正是 α-β 剪枝统一用 α ≥ β 作为剪枝条件的原因。


【总结】
AB剪枝优化的MiniMax搜索包含【两个并行发生、方向相反的流程】：
向下传递：调用子节点时，α 和 β 作为参数传入（当前节点只更新自己负责的那个边界，另一个原样传递）。
向上返回：子节点返回后，当前节点根据返回值更新自己的 α 或 β。
循环内更新：这个更新后的 α 或 β，会作为下一个兄弟节点调用时的参数（与另一个边界一起）继续向下传。
所以整个搜索过程中，α 和 β 是在【【不断收紧的窗口】】———— α一次次上抬作为初态传递，β永远作为正无穷传入，慢慢向下更新，卡出合适的更大α
它们从根节点向下传递，随着每个节点的返回值而逐步更新，更新后的值又在同层的后续兄弟节点（for循环遍历）中被用于剪枝判断。
